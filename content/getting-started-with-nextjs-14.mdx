Next.js 14 ships the App Router as stable, which fundamentally changes how you structure a project. If you've been writing Pages Router apps, a few mental models need updating.

## The biggest shift: layouts

In the Pages Router, every page was a standalone file that imported its own wrappers. In the App Router, `layout.tsx` files nest automatically — any `layout.tsx` wraps every route at or below its directory.

```tsx
// src/app/layout.tsx — wraps every page in the app
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Header />
        <main>{children}</main>
        <Footer />
      </body>
    </html>
  )
}
```

You never have to remember to add the header. It's just there.

## Server components by default

Every component in the `app/` directory is a React Server Component unless you opt out with `"use client"`. This means data fetching happens on the server, no `useEffect` needed:

```tsx
// This runs on the server — no useEffect, no loading state
export default async function PostPage({ params }: { params: { slug: string } }) {
  const post = await fetchPost(params.slug)

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.body}</p>
    </article>
  )
}
```

The component is `async`, `await` works directly inside it, and React streams the result to the client. It's a genuinely different model.

## When to use `"use client"`

Only add `"use client"` when you need browser APIs or React hooks:

```tsx
"use client"

import { useState } from "react"

export function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>
}
```

Keep client components as small and leaf-level as possible. A header with a mobile menu needs `"use client"`, but its parent layout doesn't.

## File-based metadata

Instead of `<Head>` tags, you export a `metadata` object:

```ts
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "My Blog",
  description: "Writing about web development.",
  openGraph: {
    title: "My Blog",
    images: ["/og.png"],
  },
}
```

For dynamic routes, export `generateMetadata` as an async function that receives the route params.

## The short version

- `layout.tsx` files nest automatically — put shared UI there
- Components are server-first; add `"use client"` only when needed
- `async/await` works directly in server components
- Metadata is exported, not injected via `<Head>`

The App Router has a steeper initial learning curve than Pages Router, but the mental model it converges on — server by default, client where necessary — maps cleanly onto how production apps actually need to work.
